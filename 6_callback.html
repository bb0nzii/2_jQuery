CallBack함수

- 현재 효과가 100% 완료되면 콜백함수가 실행됨
- js명령문은 줄단위(인터프리터 방식 : 한 줄씩 실행시킴)로 실행됨
- 이펙트를 사용하다가 이벤트가 완료되지 않더라도 다음 코드가 실행되면 
  순서가 맞지 않는듯한 느낌과 뒤죽박죽의 느낌
  → 이런 부분을 방지하기 위해 실행되는 것이 콜백함수!

Asynchronous

1) callback : 다른 함수의 인수로 전달되는 함수
2) Asynchronous(비동기) : 다른 함수와 병렬로 실행되는 함수
3) Promises : 생성코드와 소비코드에 대한 호출이 모두 포함됨
4) Asynce/Await : asynce를 함수 앞에 붙이면  Promise를 리턴

함수가 비동기를 사용하지 않았을 때

function myDisplayer(some){
    document.getElementById("call").innerHTML = some;
}

function myFirst(){
    myDisplayer("BBONZI");
}

function mySecond(){
    myDisplayer("JINHEE");
}

myFirst();
mySecond();

→ myFirst, mySecond 두 개를 호출했는데 mySecond만 호출됨!
  비동기를 사용하지 않았기 때문에 혼란스러워짐!
